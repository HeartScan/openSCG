# Contributing to OpenSCG

Thank you for your interest in contributing! 🚀

## How to Contribute

- **Check Issues:** Look for issues labeled “good first issue” or open a new proposal.
- **Fork the Repo:** Create your own branch from `main`.
- **Make Atomic Commits:** Keep changes focused and well-documented.
- **Write Tests:** Add or update tests for your changes.
- **Open a Pull Request:** Describe your changes and reference related issues.
- **Code Review:** Be responsive to feedback and suggestions.

## Code Style
- Use clear, descriptive variable and function names.
- Write comments for non-obvious logic.
- Follow the style of existing code (PEP8 for Python, Prettier for JS/TS).

## Community
- Be respectful and collaborative.
- No drama — just quality commits.
